# 《修仙网页游戏》MVP 开发计划

## 为什么必须先做 MVP？

三个版本的问题一样：**想要一步到位，结果一步都动不了。**

- 版本1（抖音版）：画面做了但玩法是空的，玩 5 分钟就腻
- 版本2（优化版）：指出了问题但没给执行方案
- 版本3（终极版）：功能清单太长，一个人做三个月起步，大概率烂尾

**MVP 思路：先做一个 30 秒就能上手、核心循环能跑通、有爽感的最小版本。**
画面可以丑，但**必须好玩**。好玩之后再叠功能。

---

## MVP 做哪些（Scope）

### ✅ 做（MVP 核心）

| 模块 | 内容 |
|------|------|
| 开场流程 | 黑屏→"吾命由我不由天"→免费100抽弹窗→一键领取 |
| 境界系统 | 10大境界 × 10层 = 100级，有数值公式 |
| 挂机修炼 | 离线收益，每秒自动获得修为 |
| 境界突破 | 修为够了点突破，满屏金光特效 |
| 抽卡系统 | 单抽/十连，R 60% / SR 30% / SSR 9% / 绝世 1%，保底机制 |
| 战斗闯关 | 自动战斗，无限关卡，每10关大奖 |
| 背包系统 | 查看已获得物品 |
| 基础数值 | 攻击/防御/战力公式，境界加成 |
| 本地存档 | localStorage 自动保存 |
| 手机适配 | 响应式布局，触控友好 |

### ❌ 不做（MVP 之后）

- 宗门/开宗立派系统
- 装备强化/精炼
- 灵根/资质系统
- 奇遇事件系统
- 技能系统
- 副本系统
- 排行榜
- 套装系统
- 轮回转生
- 多结局

**为什么不做宗门？**
宗门本身不产生"核心循环"。核心循环是：挂机→修为→突破→战力→推关。宗门是锦上添花，不是骨架。先让骨架站稳。

---

## 核心循环设计（这是游戏好玩与否的关键）

```
挂机修炼 ──→ 获得修为 ──→ 境界突破 ──→ 战力提升
    ↑                                       │
    │                                       ↓
  更多收益 ←── 更高关卡 ←── 自动闯关 ←──────┘
                │
                ↓
           获得灵石
                │
                ↓
           抽卡获得装备 ──→ 战力进一步提升
```

**验证标准：** 打开游戏 → 等 30 秒 → 点突破 → 点闯关 → 战力提升 → 点抽卡 → 战力再提升 → 能推更远 → 有动力继续挂机。

如果这个环转不起来，加再多功能也没用。

## MVP 数值设计（具体到公式）

### 修为相关

```
每秒挂机修为 = 5 + 境界等级 × 3 + 当前层 × 0.5
境界等级 = realmIndex（凡人0，练气1，筑基2...）

突破所需修为 = 500 × (境界等级 + 1) × 当前层
例：凡人1层需要 500，凡人10层需要 5000
   练气1层需要 1000，金丹1层需要 2000
```

### 战力相关

```
玩家战力 = (攻击 × 2 + 防御) × (1 + 境界等级 × 0.5 + 当前层 × 0.05)

关卡怪物战力 = 20 × 1.15^关卡数
第1关: 23    第5关: 40    第10关: 81
第20关: 327  第30关: 1324 第50关: 21675

通关奖励灵石 = 10 + 关卡数 × 2
```

### 抽卡相关

```
单抽价格: 200 灵石（新手免费送100抽）
保底: 10抽必出SR，80抽必出SSR

卡池概率:
  R     60%
  SR    30%
  SSR   9%
  绝世  1%（红品，全屏特效）
```

## 阶段路线图

```
阶段1（1-3天）: MVP 核心循环
  ├── 境界 + 挂机 + 突破（可验证数值是否合理）
  ├── 战斗闯关（可验证战力曲线）
  ├── 抽卡系统（可验证概率和爽感）
  └── 存档 + UI 打磨

阶段2（3-5天）: 扩展核心玩法
  ├── 宗门系统（开宗立派）
  ├── 装备系统（8部位 + 强化）
  ├── 灵根资质
  └── 每日任务

阶段3（5-10天）: 丰富内容
  ├── 奇遇事件（100+ 随机事件）
  ├── 副本系统
  ├── 技能系统
  └── 排行榜

阶段4（持续）: 差异化创新
  ├── 轮回转生
  ├── 多结局分支
  ├── 传承系统
  └── 世界Boss活动
```

**每个阶段的交付物都是一个可玩的 HTML 文件，不是文档。**
